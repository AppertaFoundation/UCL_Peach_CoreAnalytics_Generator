Challenges faced and how we solved them

1) Deploying Kafka on DC/OS

One of the big challenges we faced was deploying Kafka as a running service on dc/os. Initially after installing it, one of the brokers ( producers) would immediately close. Later we identified the problem which was the lack of available available cores in our DC/OS cluster. Therefore the solution was to create a new DC/OS template with more CPU cores and storage space.

2) Accessing the Apache NiFi UI from DC/OS

We faced the difficulty connecting to the Apache Nifi UI page remotely. We assumed that the Nifi service should work as it did locally, while the fact is we cannot access it with the same way. The URL of Nifi service is hidden by a private IP address, which means we can only access it from internal network. Our solution to this problem is X11 forwarding. We used ssh to login the master virtual machine and installed a browser in the remote server. After that the terminal forwarded the UI of the browser.

Future work

For future development on the Peach Analytics Engine, there are 2 key areas which can be further extended upon for the project.



1) Deploying Metabase/Druid on DC/OS.

During our project, we discovered that there is no framework or services to install Druid on DC/OS like we did with NiFi, Kafka and Spark. Therefore there is a lot of work to be done in order to replicate the Visualisation Tool from our local workflow onto the cloud service.

2) Implementing Kafka Streams and optimizing its processing speed

Another component that is important to the PEACH infrastructure is the Kafka Streams. Once its installed, the processing capabilities of the data from NiFi to Kafka can be greatly improved, resulting in more faster and efficient analysis.
