Testing Strategies

Core Analytics

Unit Testing: Since the PEACH system consists of multiple interconnected components running as a service, there are a very few areas that can be tested separately

Area 1: Apache Kafka

This is a very important component of the PEACH dataflow and Kafka should be tested to see if it outputs the expected type of data.

Area 2: Connection between NiFi and Kafka

The interaction between NiFi and Kafka can also be tested to view the transformation of data before it enters Kafka. (Decryption, Compression, etc.)

Performance Testing: The performance of the system can be tested by measuring its processing time. This will allow us to experiment and optimise the Kafka streams as necessary.

The following table can be used to record the results of files with different file sizes and types

Generator

Unit Testing: There are 2 main areas components in the Generator that can be tested separately, the GUI and Non – GUI elements.

Area 1 : Non – GUI elements

  • Different test cases can be written for different segments of code
  • Run the test to ensure it fails
  • Adjust the code to pass the test
  • Look for unintentional side affects

For example: A test can written so that it checks whether the data is correctly written onto a new file

Area 2:  GUI elements

  • TestFx can used to automate user interface testing
  • Different methods can be tested separately to ensure they perform the correct functions

For example: The RadioButton object can tested to check its behaviour.

Functional Testing: The main purpose of functional testing will be to test the generation of new data from novel datasets.

  • This includes identifying correct information that needs to be replaced with
  • Identifying correct dependencies within fields
  • Identifying correct variable types
