Topic 1: DC/OS

DC/OS  – What is it?

DC/OS is a distributed operating system based on the Apache Mesos distributed systems kernel. It enables the management of multiple machines as if they were a single computer. It automates resource management, schedules process placement, facilitates inter-process communication, and simplifies the installation and management of distributed services. Its included web interface and available command-line interface (CLI) facilitate remote management and monitoring of the cluster and its services.

Why are we using it?

We chose DC/OS as the main cluster manager for our resources because it is very compatible with Azure Stack and contains common useful management interfaces such as a GUI. This allows the users to easily monitor and debug problems thus reducing downtime. Furthermore , it supports sense discovery and package management.

Topic 2: Apache Kafka

Kafka  – What is it?

Kafka is used for building real-time data pipelines and stream and stream apps. It is horizontally scalable, fault-tolerant, and wicked fast.

Why are we going to use it?

It lets us publish and subscribe to streams of records. In this respect it is similar to a message queue or enterprise messaging system.

It lets us store streams of records in a fault-tolerant way.

It lets us process stream of records as they occur.

We are going to use Kafka as a data hub of our architecture, receiving the data stream from Nifi and interacting with other analytic frameworks such as spark and druid.

Topic 3: Apache Spark

Spark  – What is it?

Apache Spark is a fast and general engine for large-scale data processing, also advertised as ‘lightning fast open source cluster computing’. Spark is Apache’s most active project at the moment, providing software developers with an application program interface that allows them to run programs quickly. Spark also enables faster and more efficient code writing as it has over 80 high-level operators at the users’ disposal

Why are we going to use it?

Spark provides a framework for advanced analytics which includes tools for accelerating queries, a machine learning library, a graph processing engine and a streaming analytics engine.

Spark accelerates results as it provides parallel in-memory processing that returns results much faster than any other approach that requires disk access.

It is easy and fast to use as it has several prebuilt libraries.

Topic 4: Apache NiFi

NiFi  – What is it?

Apache Nifi is a real time integrated data logistics and simple event processing platform that automates the movement of data between disparate data sources and systems, making data ingestion fast, easy and secure.

Why are we going to use it?

It is an effective platform for building powerful and scalable data-flows.

It lends well to visual creation and management of directed graphs of processors.

It tracks the point at which data enters and exits the system well.

Will allow us to pre-process the radiology data to work well with Druid and observe its metadata.

Topic 5: Druid

Druid  – What is it?

Druid is an open-source analytics data store designed for business intelligence (OLAP) queries on event data. Druid provides low latency (real-time) data ingestion, flexible data exploration, and fast data aggregation.” Druid was originally inspired by existing data stores such as Google’s PowerDrill and it stores data in columns which is suitable for querying and filtering.

Why are we going to use it?

Druid has some outstanding key features that makes it perfect for fast data aggregation. These would be:

1) Sub OLAP Queries : Druid’s column structure and indexing enables multi-layer filtering and scanning which is perfect for querying.

2) Real-time data ingestion : Druid allows rapid ingestion of heavy data sets as much as upto “10,000+ events” per second.

3) Cost-Effective : Druid is not only open source but has numerous features built in for cost reduction.

4) Scalable : " Existing Druid deployments handle trillions of events, petabytes of data, and thousands of queries every second."

Furthermore Druid is suitable for our project because the PEACH infrastructure requires fast aggregation of heavy data sets ( lots and lots of data) and can it be further worked on for future dashboard applications.
